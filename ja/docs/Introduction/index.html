イントロダクション

IFC.js は、IFC モデルをブラウザで読み込み、表示、編集するための JavaScript ライブラリです。

🎮 あなたの IFC モデルで、ライブデモ を試してみましょう。

🔍 IFC.js について、5 分以内で 見ていきましょう。

このライブラリはどのようなものか?#

IFC とは?#

🏚 建築家は、建物の設計図を一枚ずつ描くのではなく、 The SIMS でモデル化された住宅のように、 建物の 3D モデルを作成します。 作成されたモデルには形状と情報の両方が含まれているため、 この方法は BIM（Building Information Model）と呼ばれています。

🖇 しかし、建物は非常に複雑であり、これらのモデルをすべて作成できるアプリケーションは存在しません。 それぞれのアプリケーションには、形状のモデル化、構造の計算、エネルギーシミュレーション、 プロジェクトドキュメントの作成など、それぞれの特化した機能があります。

⚠ これらのツールは、世界中の複数の開発者から提供されており、それぞれが独自のフォーマットで動作しています。 つまり、構造計算アプリケーションが作成した構造モデルは、エネルギーシミュレーションソフトでは読めません。その逆も同様です。 では、どのようにして複数のツールから建物のモデルを作ることができるのでしょうか？

🎉 その答えは IFC です。 BuildingSMART 社が作成した、建物の 3D モデルを格納するためのフォーマットです。 オープンなので、誰でも読み書きできます。最高でしょ！！

💡

IFC のおかげで、開発者は誰でも自由に BIM アプリを作ることができ、 建設分野の大手開発会社のアプリと連携することができます。

IFC の問題#

BIM アプリケーションを作ろうとする人は、他のツールから情報をインポート・エクスポートできるように IFC ファイルを読み書きする必要があります。 しかし残念ながら、IFC は読み書きが難しいフォーマットです。 何千ページものドキュメントがあり、自作の IFC パーサーを作成・維持するのは大変な作業です。

💸 IFC で開発する余裕ありますか?

これは、何人もの開発者がフルタイムで働いている企業でなければできないことです。 これではもうそんなにオープンな感じではないですよね。

IFC の内部がどのようになっているか気になる方のために、例を挙げてみましょう。 このような何十万行ものデータを解析することを想像してみてください。

#6699= IFCCARTESIANPOINT((0.,-1.70530256582424E-13));
#6701= IFCAXIS2PLACEMENT2D(#6699,#23);
#6703= IFCCARTESIANPOINT((892.,-253.399999999998,150.));
#6705= IFCAXIS2PLACEMENT3D(#6703,#15,#19);
#6706= IFCEXTRUDEDAREASOLID(#6702,#6705,#19,506.8);
#6707= IFCCOLOURRGB($,0.501960784313,0.501960784313,0.501960784313);

💀 パーサーを 1 から作り、IFC スキーマが変更されるたびにメンテナンスする気があったとしても、 この問題に取り組むにはいくつかの自明ではない問題があります。 非常に大きなファイルのメモリをどのように管理するか？ 幾何学的な生成をいかに効率的に実装するか？ IFC が正しく定義されていない場合はどうすればよいか？

IFC.js による手助け#

ジオメトリやデータのインポート／エクスポートという点では同じなのに、 すべての開発者がそれぞれのアプリケーションに IFC ファイルのリーダー／ライターを実装しなければならないというのは意味がありません。

IFC.jsは、IFCファイルの読み書きを非常に簡単に行うことができるJavaScriptライブラリです。

これにより、建築・建設アプリケーションの開発者は、 IFC での作業が容易になり、ビジネスに真に付加価値を与える機能性に集中することができます。

IFC.js の簡単な説明#

IFC.js は IFC ファイルを読み込んで、2 つのものを作成します。

🧊 ジオメトリ

IFC.js が 3D シーンを生成できるのは、Three.js や Babylon.js などの 3D ライブラリに対応しているからです。 つまり、3D の BIM ツールをすぐに作ることができるのです。

🔗 データ

ジオメトリに関連するすべてのプロパティにハイレベルでアクセスできます。 つまり、建築物の構成要素やその材料、熱特性、構造強度などのデータに簡単にアクセスできます。

IFC.js を使った BIM アプリケーションの作成は、JavaScript、HTML、CSS で Web ページを作成するのと同じくらい簡単です。IFC ファイルがない場合は、これらのファイルを読み込んでみてください。

import { IfcLoader } from 'web-ifc-three';
import { Scene } from 'three';
//THREE.js のシーンを作成
const scene = new Scene();
...
//IFC を読み込んでシーンに追加
const ifcLoader = new IfcLoader();
ifcLoader.load(ifcURL, (geometry) => scene.add(geometry));

誰に向けたライブラリー？#

このライブラリは、BIM アプリケーションを開発しようとする人、 つまり建設業界向けのアプリケーションを作成する開発者と、 IFC ファイルに含まれるデータを活用したいと考えている建築家やその他の専門家の両方を対象としています。

このライブラリを使用するには、JavaScript、HTML、CSS の基本的な知識が必要となります。 この知識は、このドキュメントの範囲を超えています。 また、IFC のジオメトリを表示するために、何らかの 3D ライブラリ（Three.js または Babylon.js）を使用する必要があります。 どこから手をつけていいかわからない場合は、こちらをご覧になるか、 Discord チャンネルに参加して気軽に聞いてみてください。

IFC.js はなぜ違うの？#

IFC.js は、最も広く使われている言語の一つである JavaScript のために開発されたライブラリです。 つまり、Web ブラウザ、デスクトップアプリケーション、モバイルアプリケーションとの互換性があります。 また、JavaScript は最も簡単に習得できる言語の一つであり、HTML や CSS を使ってユーザーインターフェースを構築することができます。 言い換えれば、IFC.js を使った BIM アプリケーションの作成は、ウェブページを作成するのと同じくらい簡単なのです。

マルチプラットフォーム#

IFC.js を使って、あらゆるプラットフォームに対応したオープンな BIM アプリケーションを作ることができます。

  • ウェブ: IFC ファイルを読み書きし、サーバー通信に依存せずに 3D を表示するフロントエンドの Web アプリケーションは、 バニラ JavaScript や、React、Vue、Angular、Svelte などのツールを使って作成できます。 つまり、IFC.js を使えば、どんな Web ブラウザもオープンな BIM アプリケーションに変えることができるのです。 ブラウザで IFC を処理して 3D を表示するには、ネイティブアプリケーションとしては力不足だと思いますか？ IFC.js を使用した公式の Three.js の例で、ご自身の目で確かめてください。

  • サーバー: クライアント側で IFC ファイルを処理できない場合があります。 例えば、モバイル端末では、中規模または大規模な IFC を複数表示するパワーがない場合があります。 このような場合、Node.js を介してサーバー上で IFC.js を使用し、 すでに用意されている Three.js のシーンをクライアント側に送信することが可能です。

  • デスクトップ: Web アプリケーションではなく、Windows や iOS で動作する ネイティブなデスクトップアプリケーションを作りたいと思うことがあります。 これは、Electron のような技術のおかげで可能になりました。 これらのネイティブアプリケーションは、ウェブアプリケーションと同じように、 HTML、CSS、JavaScript、React、Vue などを使ってプログラムされます。

  • モバイル: IFC.js は、WebGL コードをネイティブの OpenGL にマッピングする React Native にも対応しています。 つまり、IFC 対応の Open BIM アプリを Android や iOS で動作するように、 追加の手間をかけずに作成することができるのです。

スピード ⏱️#

Web アプリケーションの経験がある方は、JavaScript ベースの IFC ライブラリの弱点は パフォーマンスだと思っているかもしれません。 しかし、IFC.js のコア部分は、WebAssembly と Emscripten を組み合わせた C++で開発されています。 これは最大限のパフォーマンスを得るために特別に設計されたもので、 IFC.js はブラウザ内で直接実行しながら、デスクトップアプリケーションの速度とパフォーマンスに近づけることができます。

機能#

現状 IFC.js は以下ができます。

  • IFC ファイルの読み込み

  • 3D のジオメトリの作成

  • ジオメトリの表示の編集

  • 複数のモデルの読み込み

  • IFC のプロパティの取得

  • IFC ファイルの書き出し

私達は今、以下のことができるように開発しています。

  • あらゆる種類のジオメトリの読み込み

  • 複数の大容量ファイルを 60fps で扱う

ソースコード#

IFC.js は、大規模な多言語ライブラリ（C++、TypeScript、JavaScript など）です。 これを 1 つのリポジトリに収めるのは難しく、扱いにくいものです。 そのため、このライブラリは複数のリポジトリに存在し、それぞれが独自の目的を持っています。

  • web-ifc はライブラリのコアを含むリポジトリです。 C++でスクラッチで書かれたパーサーとジオメトリジェネレータを Emscripten 経由で WebAssembly にコンパイルしています。 このリポジトリは、IFC ファイルを読み込んでそのデータをメモリにロードするという複雑な作業をカプセル化しています。

  • web-ifc-three は、web-ifc を Three.js のジオメトリの形式に適合させるライブラリです。 このライブラリは Three.js の公式 IFC ローダーです。 このアダプターのおかげで、Three.js を使ったオープンな BIM アプリケーションをわずか 2 行のコードで作成することができます。

  • web-ifc-babylonは、このライブラリを babylon.js 用にアレンジしたものです。

  • web-ifc-viewer は、ブラウザベースの IFC ビューアであり、 IFC.js と three.js でできることの多くの例を集めたボイラープレート・ライブラリです。 すべての BIM アプリケーションには、シーンの操作、材料変更、クリックによる要素選択、断面図などの共通の機能性がありますです。 このリポジトリには、これらすべての機能性の例があるので、オープンな BIM アプリケーションですぐに再利用することができます。 ここで試すことができます。

コントリビュート#

IFC.js でオープン BIM 開発の世界をどんどん良くしていきたいと思いませんか？ ライブラリは大きいので、どこに参加したいかを選択する必要があります。

パーシングとジオメトリとC++

もしあなたが C++ や WebAssembly、パーシングや IFC 規格の低レベル実装に情熱を持っているなら、 web-ifc で私たちを助けることができます。 これは最大かつ最も複雑なライブラリです。

Three.js とジオメトリと Typescript

一方、Three.js の開発者の方でライブラリに貢献したいと考えている方は、 Three.js の公式 IFC ローダーである web-ifc-three で協力することができます （ジオメトリの最適化、IFCローダーAPIの構築など）。

BIM ツールとブレインストーミングと UX

最後に、もしあなたが Open BIM アプリケーションの構築にしか興味がなく、 BIM ツールやユーザーインターフェースについて素晴らしいアイデアをお持ちなら、 web-ifc-viewer を市場で最高の IFC ビューアにするために私たちに協力していただくのが一番です。

いずれにしても、Discord チャンネルに立ち寄っていただき、 気軽にあなたの考えを教えていただければ、私たちがご案内いたします。
