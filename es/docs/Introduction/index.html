Introducción

🎮 IFC.js es una librería de JavaScript para cargar, visualizar y editar modelos IFC en el navegador. Prueba uno de los ejemplos con tu modelo IFC y descubre la potencia de IFC.js.

¿Qué es esta librería?#

¿Qué es IFC?#

🏠 Los arquitectos ya no dibujan los planos de los edificios uno a uno, sino que crean modelos en 3D de los mismos, como las casas modeladas en Los SIMS. Esta forma de trabajar se llama BIM (Building Information Model), porque los modelos creados contienen tanto geometría como información.

🔥 Sin embargo, un edificio es tan complejo que no hay ninguna aplicación que pueda crear estos modelos en su totalidad. Cada aplicación tiene una misión específica: modelar la geometría, calcular las estructuras, hacer simulaciones energéticas, crear documentos de proyecto, etc.

👀 Estas herramientas proceden de múltiples desarrolladores de todo el mundo y cada una trabaja en su propio formato. En otras palabras, el modelo estructural creado por una aplicación de cálculo estructural no puede ser leído por un software de simulación energética, y viceversa. Entonces, ¿cómo es posible crear un modelo de un edificio a partir de múltiples herramientas?

🎉 La respuesta es IFC, un formato creado por BuildingSMART para contener estos modelos 3D de edificios. Es abierto, por lo que cualquiera puede leerlo y escribirlo. Sí, es un formato abierto, por lo que cualquiera puede leerlo y escribirlo.

💡
Gracias a IFC cualquier desarrollador puede crear libremente aplicaciones BIM que se comuniquen con las aplicaciones del gran desarrollo companies in the construction sector.

El problema de IFC#

Cualquiera que quiera hacer una aplicación BIM necesita leer y escribir archivos IFC para poder importar y exportar información desde otras herramientas. Por desgracia, IFC es un formato difícil de leer y escribir. Hay miles de páginas de documentación y crear y mantener un parser IFC hecho por uno mismo es una tarea enorme.

¿Puedes permitirte usar IFC?

💸 Esto es algo que sólo pueden permitirse empresas con varios desarrolladores trabajando a tiempo completo en esto. No parece tan abierto, ¿verdad?

Por si tienes curiosidad por saber cómo es un IFC por dentro, aquí tienes un ejemplo. Imagina analizar cientos de miles de líneas como estas:

#6699= IFCCARTESIANPOINT((0.,-1.7053025E-13));
#6701= IFCAXIS2PLACEMENT2D(#6699,#23);
#6703= IFCCARTESIANPOINT((892.,-253.399999,150.));
#6705= IFCAXIS2PLACEMENT3D(#6703,#15,#19);
#6706= IFCEXTRUDEDAREASOLID(#6702,#6705,#19,506.8);
#6707= IFCCOLOURRGB($,0.50196,0.501913,0.501960);

💀 Incluso si estás dispuesto a crear un analizador sintáctico desde cero y mantenerlo con cada cambio en el esquema IFC, hay varias cuestiones no triviales para abordar este problema. ¿Cómo gestionar la memoria en archivos muy grandes? ¿Cómo implementar la generación geométrica de forma eficiente? ¿Qué hacer cuando un IFC no se ha definido correctamente?

IFC.js al rescate#

No tendría sentido que todos los desarrolladores tuvieran que implementar su propio lector y escritor de archivos IFC para sus aplicaciones . Especialmente cuando todos hacemos lo mismo: importar y exportar geometría y datos.

IFC.js es una librería de JavaScript que hace que sea súper fácil leer y escribir archivos IFC.

Esto permite a los desarrolladores de aplicaciones de arquitectura y construcción trabajar en IFC sin esfuerzo y centrarse en las funcionalidades que realmente añaden valor a su negocio.

IFC.js en pocas palabras#

IFC.js tiene dos enfoques:

💎

Geometría

IFC.js es capaz de generar escenas 3D porque es compatible con librerías 3D como Three.js o Babylon.js. Esto significa que puedes crear herramientas BIM 3D sin esfuerzo.

🔗

Datos

Acceso de alto nivel a todas las propiedades asociadas a esa geometría. Esto significa un fácil acceso a los datos de los elementos constructivos, sus materiales, características térmicas, resistencia estructural, etc.

Crear una aplicación BIM con IFC.js es tan fácil como crear una página web con JavaScript, HTML y CSS. Si no tienes ningún archivo IFC disponible, prueba a cargar cualquiera de estos.

import { IfcLoader } from 'web-ifc-three';
import { Scene } from 'three';
//Crea la escena THREE.js
const scene = new Scene();
//...
//Carga el IFC y lo añade a la escena
const ifcLoader = new IfcLoader();
ifcLoader.load(ifcURL, (geometry) => scene.add(geometry));

Para quién es esta librería#

Esta librería está dirigida a cualquier persona que quiera desarrollar aplicaciones BIM. Esto incluye tanto a los desarrolladores que crean aplicaciones para el sector de la construcción como a los arquitectos y otros profesionales que quieren aprovechar los datos de sus archivos IFC.

Para utilizar esta librería es necesario tener un conocimiento básico de JavaScript, HTML y CSS. Estos conocimientos están fuera del alcance de esta documentación. También necesitarás utilizar alguna librería 3D para mostrar la geometría del IFC (Three.js o Babylon.js).

😊 Si no sabes por dónde empezar, te sugerimos que eches un vistazo aquí o que te unas al Canal de Discord y nos saludes.

¿Por qué es diferente IFC.js?#

IFC.js es una librería escrita por y para JavaScript, posiblemente uno de los lenguajes más ubicuos. Esto significa que es compatible con los navegadores web y las aplicaciones de escritorio y móviles. Además, JavaScript es uno de los lenguajes más fáciles de aprender y permite construir interfaces de usuario con HTML y CSS. En otras palabras, crear aplicaciones BIM con IFC.js es tan fácil como crear una página web.

Multiplataforma#

Puedes utilizar IFC.js para crear aplicaciones BIM abiertas para cualquier plataforma:

  • 🌏
  • 🏢
  • 💻
  • 📱

Se pueden crear aplicaciones web frontend que lean y escriban archivos IFC y muestren 3D sin depender de la comunicación con el servidor utilizando Vanilla JavaScript u otras herramientas como React, Vue, Angular, Svelte, etc. Es decir, con IFC.js podemos convertir cualquier navegador web en una app BIM abierta.

Velocidad#

⚡️ Si tienes experiencia con aplicaciones web, podría pensar que el punto débil de una librería IFC basada en JavaScript es el rendimiento. Sin embargo, el núcleo de la librería está desarrollado en C combinado con WebAssembly y Emscripten. Está diseñado específicamente para obtener el máximo rendimiento, lo que permite que IFC.js se acerque a la velocidad y el rendimiento de las aplicaciones de escritorio mientras se ejecuta directamente en el navegador.

Características

👓 IFC.js puede leer archivos IFC sin problemas.

🏐 Genera geometría 3D que se ejecuta a 60 fps en un navegador.

💄 Edita fácilmente la apariencia de la geometría con el poder de Three.js.

🏠🏠🏠 Cargar múltiples modelos IFC federados.

📃 Recuperar propiedades IFC para informes y bases de datos.

✍ IFC.js también puede editar y escribir archivos IFC desde cero.

Código de la fuente

IFC.js es una librería grande y multilingüe (C , TypeScript JavaScript, etc). Tenerla en un único repositorio sería difícil y poco manejable. Por esa razón, la librería existe en múltiples repositorios; cada uno con un único propósito. Puedes encontrarlos aquí.

  • web-ifc
  • web-ifc-three
  • web-ifc-viewer

El núcleo de la librería: un parser de IFC y generador de geometría escrito desde cero en C y compilado mediante Emscripten a WebAssembly. Este repositorio encapsula la complejidad de la lectura de archivos IFC y la carga de sus datos en la memoria.
