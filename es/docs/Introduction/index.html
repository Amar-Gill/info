Introduction

🎮 IFC.js es una biblioteca de JavaScript para cargar, mostrar y editar modelos IFC en el navegador. Pruebe la demostración en vivo con su modelo IFC y descubra la potencia de IFC.js.

¿Qué es esta biblioteca?#

¿Qué es la CFI?#

🏠 Los arquitectos ya no dibujan los planos de los edificios uno a uno, sino que crean modelos en 3D de los mismos, como las casas modeladas en Los SIMS. Esta forma de trabajar se llama BIM (Building Information Model), porque los modelos creados contienen tanto geometría como información.

🔥 Sin embargo, un edificio es tan complejo que no hay ninguna aplicación que pueda crear estos modelos en su totalidad. Cada aplicación tiene una misión específica: modelar la geometría, calcular las estructuras, hacer simulaciones energéticas, crear documentos de proyecto, etc.

👀 Estas herramientas proceden de múltiples desarrolladores de todo el mundo y cada una trabaja en su propio formato. En otras palabras, el modelo estructural creado por una aplicación de cálculo estructural no puede ser leído por el software de simulación energética, y viceversa. Entonces, ¿cómo es posible crear un modelo de un edificio a partir de múltiples herramientas?

🎉 La respuesta es IFC, un formato creado por BuildingSMART para contener estos modelos 3D de edificios. Es abierto, por lo que cualquiera puede leerlo y escribirlo. Sí, es un formato abierto, por lo que cualquiera puede leerlo y escribirlo.

💡

Gracias a IFC, cualquier desarrollador puede crear libremente aplicaciones BIM que se comuniquen con las aplicaciones de las grandes empresas promotoras del sector de la construcción.

El problema de la CFI#

Cualquiera que quiera hacer una aplicación BIM necesita leer y escribir archivos IFC para poder importar y exportar información de otras herramientas. Por desgracia, IFC es un formato difícil de leer y escribir. Hay miles de páginas de documentación y crear y mantener un parser IFC hecho por uno mismo es una tarea enorme.

¿Puedes permitirte la CFI?

💸 Esto es algo que sólo pueden permitirse empresas con varios desarrolladores trabajando a tiempo completo en esto. Ya no suena tan abierto, ¿verdad?

Por si tienes curiosidad por saber cómo es un IFC por dentro, aquí tienes un ejemplo. Imagina analizar cientos de miles de líneas como estas:

#6699= IFCCARTESIANPOINT((0.,-1.7053025E-13));
#6701= IFCAXIS2PLACEMENT2D(#6699,#23);
#6703= IFCCARTESIANPOINT((892.,-253.399999,150.));
#6705= IFCAXIS2PLACEMENT3D(#6703,#15,#19);
#6706= IFCEXTRUDEDAREASOLID(#6702,#6705,#19,506.8);
#6707= IFCCOLOURRGB($,0.50196,0.501913,0.501960);

💀 Incluso si estás dispuesto a crear un analizador sintáctico desde cero y mantenerlo con cada cambio en el esquema IFC, hay varias cuestiones no triviales para abordar este problema. ¿Cómo gestionar la memoria en archivos muy grandes? ¿Cómo implementar la generación geométrica de forma eficiente? ¿Qué hacer cuando un IFC no se ha definido correctamente?

IFC.js al rescate#

No tendría sentido que todos los desarrolladores tuvieran que implementar su propio lector y escritor de archivos IFC para sus aplicaciones cuando todos hacemos lo mismo: importar y exportar la geometría y los datos.

IFC.js es una biblioteca de JavaScript que hace que sea muy fácil leer y escribir archivos IFC.

Esto permite a los desarrolladores de aplicaciones de arquitectura y construcción trabajar en IFC sin esfuerzo y centrarse en las funcionalidades que realmente añaden valor a su negocio.

IFC.js en pocas palabras#

IFC.js lee los archivos IFC y genera dos cosas:

🧊 Geometría

IFC.js es capaz de generar escenas 3D porque es compatible con bibliotecas 3D como Three.js o Babylon.js. Esto significa que puede crear herramientas BIM en 3D de inmediato.

🔗 Datos

Acceso de alto nivel a todas las propiedades asociadas a esa geometría. Esto significa un fácil acceso a los datos de los componentes del edificio, sus materiales, características térmicas, resistencia estructural, etc.

Crear una aplicación BIM con IFC.js es tan fácil como crear una página web con JavaScript, HTML y CSS. Si no tienes ningún archivo IFC disponible, prueba a cargar cualquiera de estos.

import { IfcLoader } from 'web-ifc-three';
import { Scene } from 'three';
//Crea la escena THREE.js
const scene = new Scene();
//...
//Carga el IFC y lo añade a la escena
const ifcLoader = new IfcLoader();
ifcLoader.load(ifcURL, (geometry) => scene.add(geometry));

Para quién es esta biblioteca#

Esta biblioteca está dirigida a cualquier persona que quiera desarrollar aplicaciones BIM: tanto a los desarrolladores que crean aplicaciones para el sector de la construcción como a los arquitectos y otros profesionales que quieren aprovechar los datos contenidos en sus archivos IFC.

Para utilizar esta biblioteca es necesario tener un conocimiento básico de JavaScript, HTML y CSS. Estos conocimientos están fuera del alcance de esta documentación. También necesitarás utilizar alguna librería 3D para mostrar la geometría del IFC (Three.js o Babylon.js). Si no sabes por dónde empezar, te sugerimos que eches un vistazo aquí o simplemente únete al Canal de Discord y saluda.

¿Por qué es diferente IFC.js?#

IFC.js es una biblioteca escrita por y para JavaScript, posiblemente uno de los lenguajes más ubicuos. Esto significa que es compatible con los navegadores web y las aplicaciones de escritorio y móviles. Además, JavaScript es uno de los lenguajes más fáciles de aprender y permite construir interfaces de usuario con HTML y CSS. En otras palabras, crear aplicaciones BIM con IFC.js es tan fácil como crear una página web.

Multiplataforma#

Puede utilizar IFC.js para crear aplicaciones BIM abiertas para cualquier plataforma:

  • Web: se pueden crear aplicaciones web frontales que lean y escriban archivos IFC y muestren 3D sin depender de la comunicación con el servidor utilizando vanilla JavaScript u otras herramientas como React, Vue, Angular, Svelte, etc. Es decir, con IFC.js podemos convertir cualquier navegador web en una aplicación BIM abierta. Crees que un navegador no es lo suficientemente potente como para procesar un IFC y mostrar el 3d como una aplicación nativa? Puedes comprobarlo tú mismo en el ejemplo oficial de Three.js, que utiliza IFC.js.

  • Servidor**: Hay casos en los que el archivo IFC no puede ser procesado en el cliente. Por ejemplo, es posible que un dispositivo móvil no tenga la potencia necesaria para mostrar varios IFC de tamaño medio o grande. En este caso es posible utilizar IFC.js en el servidor a través de Node.js y enviar la escena Three.js ya preparada al cliente.

  • Escritorio: A veces no queremos crear aplicaciones web, sino aplicaciones nativas de escritorio que se ejecuten en Windows o iOS. Esto es posible gracias a tecnologías como Electron. Estas aplicaciones nativas se programarán de la misma manera que las aplicaciones web; HTML, CSS, JavaScript, React, Vue, etc.

  • Móviles: IFC.js también es compatible con React Native, que mapea el código WebGL a OpenGL nativo. Esto significa que puedes crear aplicaciones Open BIM compatibles con IFC que se ejecuten en Android e iOS sin ningún esfuerzo adicional.

Velocidad#

⏱️ Si tiene experiencia con aplicaciones web, podría pensar que el punto débil de una biblioteca IFC basada en JavaScript es el rendimiento. Sin embargo, el núcleo de la biblioteca está desarrollado en C combinado con WebAssembly y Emscripten. Está diseñado específicamente para obtener el máximo rendimiento, lo que permite a IFC.js acercarse a la velocidad y el rendimiento de las aplicaciones de escritorio mientras se ejecuta directamente en el navegador.

Características

Actualmente IFC.js puede:

  • Leer archivos IFC

  • Generar geometría 3D

  • Editar la visualización de la geometría

  • Cargar varios modelos

  • Recuperar las propiedades del IFC

  • Escribir archivos IFC

Actualmente estamos trabajando para poder:

  • Cargar cualquier tipo de geometría

  • Maneja múltiples archivos grandes a 60 fps

Código de la fuente#

IFC.js es una biblioteca grande y multilingüe (C , TypeScript JavaScript, etc). Tenerla en un único repositorio sería difícil y poco manejable. Por esa razón, la biblioteca existe en múltiples repositorios, donde cada uno tiene su propio propósito:

  • web-ifc es el repositorio que contiene el núcleo de la librería: un parser y un generador de geometría escritos desde cero en C y compilados mediante Emscripten a WebAssembly. Este repositorio encapsula la complejidad de la lectura de los archivos IFC y la carga de sus datos en la memoria.

  • web-ifc-three es la librería que adapta web-ifc a las entidades geométricas de Three.js. Esta librería es el cargador IFC oficial de Three.js.Gracias a este adaptador se pueden crear aplicaciones BIM abiertas con Three.js en sólo dos líneas de código.

  • web-ifc-babylon es la adaptación de esta biblioteca para babylon.js.

  • web-ifc-viewer es un visor IFC basado en el navegador y una librería boilerplate con muchos ejemplos de lo que se puede hacer con IFC.js y three.js. Todas las aplicaciones BIM tienen funcionalidades comunes: navegación por la escena, cambios de material, selección de elementos haciendo clic sobre ellos, planos de sección, etc. Hay un ejemplo de todas estas funcionalidades en este repositorio, por lo que se puede reutilizar en aplicaciones BIM abiertas desde el principio. Puedes probarlo aquí](https://agviegas.github.io/web-ifc-viewer/examples/simple-html/).

Contribuir#

¿Quieres ayudarnos a seguir mejorando el mundo del desarrollo de Open BIM con IFC.js? La biblioteca es grande, así que tendrás que elegir dónde quieres participar.

Parsing, geometría y C

Si te apasiona el C , WebAssembly, el parsing o la implementación de bajo nivel del estándar IFC, puedes ayudarnos conweb-ifc. Esta es la biblioteca más grande y compleja de todas.

Tres.js, geometría y Typescript

Por otro lado, si eres un desarrollador de Three.js y te gustaría contribuir a la biblioteca, puedes ayudarnos con web-ifc-three, que es el cargador IFC oficial de Three.js (optimización de la geometría, construcción de la API del cargador IFC, etc).

Herramientas BIM, brainstorming y UX

Por último, si sólo te interesa construir aplicaciones BIM abiertas y tienes grandes ideas para herramientas BIM o interfaces de usuario, lo mejor es que nos ayudes a hacer de web-ifc-viewer el mejor visor IFC del mercado.

En cualquier caso, te animamos a que te pases por el canal de Discord, nos saludes y nos cuentes lo que te preocupa para que podamos orientarte.
